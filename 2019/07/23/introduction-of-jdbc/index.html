JDBC简介

简介

JDBC是Java Database Connectivity的缩写，它是连接Java程序和数据库服务器的纽带。Java程序通过JDBC API来数据库，它的优点是：

  • 简化访问数据库的程序代码，无须设计与数据库服务器通信的细节。
  • 不依赖于任何数据库平台，同一个Java程序可以访问多种数据库服务器，也就是说可以做到平台无关，write once run every where.

实现


JDBC实现包括三部分：

  • JDBC驱动管理器：java.sql.DriverManager类，由Sun公司实现，负责注册特定JDBC驱动器，以及根据特定驱动器建立与数据库的连接。
  • JDBC驱动器API：由SUN公司制定，主要接口是java.sql.Driver接口。
  • JDBC驱动器：由数据库供应商或者是其他第三方工具提供商创建，也称为JDBC驱动程序。JDBC驱动器实现了JDBC驱动器API，负责与特定的数据库连接，以及处理通信的细节，它可以注册到JDBC驱动管理器中。

从图中可以看出，SUN公司制定了两套API。

  • JDBC API：Java应用程序用它来访问数据库
  • JDBC驱动器API：当数据库供应商或者其他第三方的工具提供商为特定数据库创建JDBC驱动器时，该驱动器得实现JDBC驱动器API。

从图中可以看出，JDBC驱动器才是连接Java应用程序和特定数据库的纽带。Java应用程序如果希望访问某种数据库，必须先获得JDBC驱动器的类库，然后把它注册到JDBC驱动器中。

Java应用程序只和JDBC打交道，JDBC API依赖于DriverManager类来管理JDBC驱动器。假如应用需要向一个MySQL数据库提交一条SQL语句，DriverManager类就会委托特定的MySQL的驱动器来执行这个任务；同理，假如应用程序需要向一个Oracle数据库提交一条SQL语句，DriverManager类就会委派特定的Oracle的驱动器来执行这个任务。

JDBC API

JDBC API主要位于java.sql包中，关键的接口与类包括以下几种。

  • Driver接口和DriverManager类：前者表示驱动器，后者表示驱动管理器。
  • Connection接口：表示数据库连接
  • Statement接口：负责执行SQL语句
  • PreparedStatement接口：负责执行预准备的SQL语句。
  • CallableStatement接口：负责执行SQL的存储过程。
  • ResultSet接口：表示查询SQL语句返回的结果集。

Driver接口和DriverManager类

所有JDBC驱动器必须实现Driver接口，JDBC驱动器由数据库厂商或第三方提供。在编写数据库的Java程序时，必须把特定数据库快的JDBC驱动器的类库加入到classpath中。

DriverManager类用来建立和数据库的连接以及管理JDBC驱动器。DriverManager类的方法都是静态的：

  • registerDriver(Driver driver)：在DriverManager中注册JDBC驱动器。
  • getConnection(String url, String user, String pwd)：建立和数据库的连接，并返回表示数据库连接的Connection对象。
  • setLoginTimeOut(int seconds)：设定等待建立数据库连接的超时时间。
  • setLogWriter(PrintWriter out)：建立输出JDBC日志的PrintWriter对象。

Connection接口

Connection接口代表Java程序和数据库的连接，主要包括一下方法：

  • getMetaData()：返回数据库的元数据的DatabaseMetadata对象。元数据包含了描述数据库的相关信息，如数据库中的表的结构就是一种元数据。
  • createStatement()：创建并返回Statement对象。
  • prepareStatement()：创建并返回PreparedStatemen对象。

Statement接口

Statement提供了三个执行SQL语句的方法。

  • exexute(String sql)：执行各种SQL语句。该方法返回一个boolean类型的值，如果为true，表示执行的SQL语句具有查询结果。可以通过Statement的getResultSet()方法获得这一查询结果。
  • executeUpdate(String sql)：执行SQL的insert、update和delete语句。该方法返回一个int类型的值，表示数据库中受该SQL语句影响的记录的数目。

ResultSet接口

ResultSet接口表示select查询语句得到的结果集。调用ResultSet对象的next()方法，可以使得游标定位到结果集中的下一条记录。调用ResultSet对象的getXXX()方法，可以获得一条记录中的某个字段的值。ResultSet提供了以下常用的getXXX()方法。

  • getString(int columnIndex)：返回制定字段的String类型的值，参数为columnIndex代表字段的索引。
  • getString(String columnIndex)：返回制定字段的String类型的值，参数为columnIndex代表字段的名字。
  • getInt(int columnIndex)：返回制定字段的Int类型的值，参数为columnIndex代表字段的索引。
  • getInt(String columnIndex)：返回制定字段的Int类型的值，参数为columnIndex代表字段的名字。
  • getFloat(int columnIndex)：返回制定字段的Float类型的值，参数为columnIndex代表字段的索引。
  • getFloat(String columnIndex)：返回制定字段的Float类型的值，参数为columnIndex代表字段的名字。

ResultSet提供了getString()、getInt()和getFloat()等方法，程序应该根据字段的数据类型来决定调用哪一种getXXX()方法。此外，程序即可以根据字段的索引来制定字段，也可以通过字段的名字来制定字段。字段的索引从1开始。

总结


我理解的JDBC其实它只是一组规范，比如JDBC API和JDBC驱动API。

所有数据库驱动由数据库厂商或者第三方提供，数据库驱动必须实现了JDBC的驱动API，这样我们就可以通过标准的JDBC API调用相应的数据驱动，而不需要知道数据库驱动是怎么实现。

很多ORM框架的实现在最底层其实就是对JDBC操作的封装，因此在学好MySQL与JDBC之后再去学习其他的ORM框架也不迟。如果想成为一个好的程序员，学好数据库是必要也是必须的。
