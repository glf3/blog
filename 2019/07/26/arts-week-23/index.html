ARTS第二十三周

　　ARTS，即：每周完成一个ARTS：每周至少做一个 leetcode 的算法题、阅读并点评至少一篇英文技术文章、学习至少一个技术技巧、分享一篇有观点和思考的技术文章，至少坚持一年。（也就是 Algorithm、Review、Tip、Share 简称ARTS）。

Algorithm

本周算法题1104. 二叉树寻路

题意：


分析：

我没有想到很好的办法，于是根据题意建立了一棵树。

  • 按照题意模拟建树，树节点存父节点还有子节点的指针
    1. 无论奇数层还是偶数层，遍历顺序都是从后往前
    2. 奇数层先右节点后左节点
    3. 偶数层先左节点后右节点
  • 建树完毕之后，最终的答案就是从子节点往根节点的路径，单次查询的最坏时间复杂度为：log(N)，即完全二叉树的最大高度。
  • 假设树节点为N，总共查询次数为M，最坏时间复杂度为:O(N) + M*log(N)。如果测试用例再多一点的话，我的这种解法性能上应该会很好。
