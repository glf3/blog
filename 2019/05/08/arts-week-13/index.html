Algorithm

本周算法题516. 最长回文子序列

题意：

分析：

这个问题和上周的问题很相似，只不过子串的限定条件变成了子序列，也就是要求连续字符变成了不连续的字符，同样也可以用动态规划去解决。

  • 定义字符串为str，dp[i][j]代表的意义是从str(i,j)区间内的最长回文子字符串。
  • 初始dp[i][i] = 1，代表单个字符串是回文字符串。
  • 状态转移方程是：
    1. 当str(i) == str(j)的时候，dp[i][j] = dp[i+1][j-1] + 2
    2. 当str(i) != str(j)的时候，dp[i][j] = max(dp[i+1][j], dp[i][j-1])
    3. 可以证明，当str(i) == str(j)的时候，dp[i][j]状态转移一定是从dp[i+1][j-1]转移而来。
  • 定义dp[i][j]为从str(i,j)之间的最大回文子串，那么dp(i+1,j-1)作为子问题，子问题具有最优解，并且没有影响dp[i][j]做的选择，因此具有最有子结构。

Review

本周阅读了一篇Medium上的文章《Threads Vs Queues》

概述

作者在文章中说明了线程的一些特点与劣势，然后给出了队列与线程之间的比较，说明了用线程完成的工作，队列都可以很好的完成。

线程 VS 队列

线程在大型的项目和应用中是必须可少的，但是它带来的痛苦也是巨大的。我们花了很长的时间去解决内存管理、竞争条件、死锁、调度，joining Thread等等问题。

作者接下来说明了队列其实也可以很好的替代线程完成工作，然后从各个方面去比较线程与队列的对比，比如：

  • 单任务线程 -> 单个顺序实行的队列
  • 工作线程
    1. 如果需要同步任务的结果，使用顺序执行的队列。
    2. 如果任务之间是互相独立的，使用并发队列。
  • 线程池 -> 并发队列

使用队列的益处

  • 队列更加便宜：
    1. 队列减少了在应用内存空间中存储栈需要的内存。
    2. 在必要情况下可以完成对内核的调整。
  • 显著减少了代码的数量：
    1. 管理和调度线程的代码我们可以不用再写，因此我们可以专注于具体的业务逻辑。
  • 底层的队列软件可以为我们很好的管理线程、创建线程，这比我们写的线程相关的代码更加有效。
  • 在队列中的任务不仅仅可以被存储、监控、还有可视化，还可以通过日志去查看应用发生了什么问题
