Review

本周阅读了一篇Medium上的文章《The Path to Becoming a Software Architect》

作者在文章中说明了一下问题：

  • 谁是软件架构师
  • 架构师的主要职责是什么
  • 如何在架构师路上成长或者是制定计划成为架构师

谁是软件架构师

软件架构师是一名软件专家，他能够做出高水平设计、选择并且制定技术规范，职责包括软件编码标准，工具还有平台等。为首的一个软件架构师一般成为首席架构师。

软件架构师需要的一些特质

  • 善于沟通：因为在日常工作中，软件架构师需要和客户还有各个职位的经理以及开发者交流，而且经常参与大多数方案的制定过程，并且保证制定的方案必须对所有相关方都是可接受、有益的。因此，沟通很为重要。
  • 广而深的技术知识：架构师应该在一些技术栈上有着不错的水平，在某一个或几个技术栈上有着杰出的水平。架构师需要撰写大量的技术文档，报告还有图表。
  • 责任：架构师应该为他所做工作和做出的决定负责。如果一个开发犯下的错误需要几天的时间就能修复，但是一个架构师犯的错可能需要一年的时间才能修复。
  • 抗压力
  • 管理技巧：因为一个团队可能有不同的技术专家组成，因此学会领导一个团队很重要。
  • 分析技巧：很重要的一个能力是把抽象的问题用一些系统里面的实体对象表示出来，这些实体对象是开发者在评估、设计、或者正在开发的。

软件架构师的职责

  • 确定项目的利益相关者
  • 为项目确定商业需求还有利益相关者的需求
  • 通过需求来设计整个系统
  • 以一个高水平的视角确定系统的架构还有系统的不同组件
  • 选定系统间不同组件之间的实现方式还有组件与组件之间的通信方式
  • review设计的架构
  • code-review.
  • 撰写项目文档及其支持
  • 在公司中创建统一的开发标准
  • 在下一个系统版本发布前控制整个架构设计

如何成为一个软件架构师

首先，定义里程碑目标以实现成为软件架构师的战略目标非常重要。

  • 理解并且尝试不同的技术栈。比如现在集中在学习iOS，那么去了解Android也是必要的，尝试几种不同的编程语言，学习Python还有刷新自己的Java EE技巧。架构师应该是一个全栈工程师，具有广而深的知识面对架构师很重要。
  • 阅读文学。个人觉得，阅读文学可以让自己保持感性，这与实际工作中的需要的理性是完全不同的。阅读文学，弹吉他，都可以让自己保持感性，保持对未来的思考。
  • 找一个导师。找一个导师是很重要的，当你刚刚起步的时候，从一个专家身上得到知识会比自己乱抓、乱逛要容易很多。
  • 学习相关的课程/获得证书。去学习你认为值得的课程并且获得证书是值得的事。

总结

人们通常都是害怕一些新的东西。比如新的工作岗位，新的压力，更愿意待在舒适区里面。

当你想成为软件架构师的时候，这不是一件短时间内可以做到的事，这需要花费好几年的时间。如果你还没有明确的计划的话，那么开始做一些力所能及的事，学习一些新的知识都比待在原地更好。

因为，在IT领域的话站在原地是生活中停滞和个人羁绊的代名词。
