题意：

分析：

根据题意来解决即可，细心一点。我一次提交就通过了还是很有成就感的哈哈。

  • 首先删除字符串str前置空格，判断字符串str是否为空，为空return 0。
  • 对每一个在字符串str中的字符c进行循环枚举，有两种选择：
    1. 如果c为数字：添加进结果字符串validString中，并且标记已遇上数字。
    2. 如果c为字符：
      1. 如果已经遇上数字，退出循环。
      2. 如果没有遇上数字，并且当前没有记录正负号。
        1. 当字符c等于’+’或者是’-‘，用enum类型sign记录下符号类型。
        2. 标记当前字符串str不合法，退出循环。
      3. 如果没有遇上数字，但已经记录了正负号，那么直接退出循环。
  • 删除结果字符串validString的前置0，如果validString为空，return 0。
  • 通过Java中的BigInteger判断结果字符串validString大小是否大于整形最大值(2^31-1)即可，而不需要判断是否小于整形的最小值(-2^31)，原因为什么大家可以思考一下。如果溢出的话，通过enum类型的sign判断符号类型，return 对应的极大/小值即可。
  • 如果没有溢出，通过enum类型的sign判断符号类型，然后返回结果字符串validString对应的正/负数即可。

Share

本周分享一篇文章《初学 Java Web 开发，请远离各种框架，从 Servlet 开发》

最近在学Java Web相关的一些知识。

我从Spring开始学起，看了《Spring实战（第4版）》几个章节的内容，同时也在看《Tomcat与Java Web开发技术详解（第2版）》这一本书籍上的内容，中途看到了这一篇文章，觉得很不错，在这里分享一下。

文章的大概内容：

建议Java Web的初学者刚开始从Servlet开始学起，不要着急去学框架，学习路径如下所示：

  • 打好Java编程基础
  • 环境准备Eclipse + Tomcat
  • 了解Servlet和Filter
  • 明确Servlet和HTTP的对应关系
  • 熟知HTTP协议

总结一下

Web框架是开发者在使用某种语言编写Web应用服务端时关于架构的最佳实践。很多Web框架是从实际的Web项目抽取出来的，仅和Web的请求和响应处理有关，形成一个基础，在开发别的应用项目的时候则可以从这个剥离出来的基础做起，让开发者更关注更具体的业务问题，而不是Web的请求和响应的控制。

学习Java Web开发时，不要一开始就学习框架，先打好Java基础，从Servlet规范开始学起，慢慢熟悉Servlet规范，然后再去学习相关的框架。
